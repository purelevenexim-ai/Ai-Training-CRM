# Pure Leven Catalog Consolidation - Implementation Start

This folder contains the first implementation slice for safe catalog consolidation.

## What is included

1. `final_catalog_spec.csv`
- Canonical family definitions.
- Category placement.
- Canonical titles and handles.
- Variant policy set to `50g|100g|200g` for applicable families.
- Grade and combo policy flags.

2. `current_to_target_mapping.csv`
- Current handles from your active export mapped to canonical families.
- Variant value mapping where available.
- `archive_redirect` actions for sizes outside target set.

3. `out/*` generated by planning script
- `family_variant_coverage.csv`: per-family target coverage.
- `redirects_to_create.csv`: old URL path to new canonical URL path.
- `unresolved_mapping_rows.csv`: mapping rows that do not match source list.
- `dry_run_report.md`: execution readiness report.

4. `scripts/prepare-catalog-consolidation-plan.mjs`
- Non-destructive script.
- Reads `product_list.csv` + mapping/spec CSVs.
- Generates dry-run artifacts without mutating Shopify.

## Run

```bash
node scripts/prepare-catalog-consolidation-plan.mjs
```

## Safety notes

1. This step does not call Shopify APIs and does not mutate catalog data.
2. Do not run mutation scripts until:
- backups are complete,
- mapping is reviewed,
- redirect list is approved,
- rollback path is validated.
