## AWS SES Identity Verification

### Identity Setup (Domain-Based)

**✅ RECOMMENDED:** Verify as Domain, NOT Email Address

| Setting | Value |
|---------|-------|
| Identity Type | **Domain** |
| Domain Name | `mail.pureleven.com` |
| MAIL FROM Domain | `bounce.mail.pureleven.com` |

**Why Domain Verification:**
- Single verification covers all subdomains
- `noreply@mail.pureleven.com` ✅ works automatically
- `offers@mail.pureleven.com` ✅ works automatically
- `support@mail.pureleven.com` ✅ works automatically
- `hello@mail.pureleven.com` ✅ works automatically

### AWS SES Deliverability Settings

After domain verification, configure:

| Setting | Recommendation | Reason |
|---------|-----------------|--------|
| Virtual Deliverability Manager | OFF | Unnecessary cost |
| Track Opens & Clicks | ON | Engagement metrics |
| Optimized Shared Delivery | ON | Better deliverability |
| Dedicated IP Pool | OFF | Shared IPs sufficient |
| Tenant Management | OFF | Not multi-tenant |
| Auto Validation | OFF | Manual control preferred |

## Troubleshooting

### Error: AccessDenied on ses:SendRawEmail

**Cause:** IAM user lacks permissions  
**Fix:** Add `ses:SendRawEmail` to `pureleven_email` user policy

### Error: Domain Not Verified

**Cause:** `mail.pureleven.com` not verified in SES  
**Fix:** Complete Phase 2 (DNS records) and wait for AWS verification

### Error: DKIM Invalid

**Cause:** DNS records not propagated  
**Fix:** Check DNS with `nslookup` or `dig`, wait 24-48 hours

### Email Not Delivering

**Cause 1:** Missing SPF/DKIM/DMARC records  
**Fix:** Verify all 4 DNS record types added

**Cause 2:** SendGrid fallback not configured  
**Fix:** Add `SENDGRID_API_KEY=SG.xxxxx` to ai-engine/.env

### Plunk API 401 Unauthorized

**Cause:** Invalid API key  
**Fix:** Verify `PLUNK_API_KEY` matches database value

